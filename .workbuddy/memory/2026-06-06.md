## EGO Beukers Implementation Status (afternoon update)

**Files created:**
- `ego_beukers_model.py` — EMModule, RecurrentContextModule, EGOBeukersModel wrapper (re-implemented, not imported from EmergentIntelligentControl/)
- `ego_beukers_run.py` — Training loop + experiment runner using NeuraGEM's seq_learnDataset and Logger
- `NON_EQUATABLE_PARAMETERS.md` — Documented 6 categories of non-equatable parameters

**Bugs fixed during implementation:**
1. `softmax(dim=0)` → `dim=-1` — Critical: dim=0 on shape (1,N) gives all-1s softmax, zero gradient. Fixed.
2. First-step backward crash — Loss lacks grad_fn when EM is empty. Added `if loss.requires_grad:` guard.
3. Logger input mismatch — Was logging `state_t` instead of `next_t` as target (NeuraGEM logs targets). Fixed.

**Current working parameters:** temperature=0.2, weighted_retrieval=True, episodc_lr=1.0
- temperature=0.05 (EGO paper default) → softmax saturates → zero gradients → model cannot learn
- temperature=0.2 → gradients flow, model learns effectively
- weighted_retrieval=True allows learned state_weight (converges to ~0.72 state weighting)

**Single-seed results (seed 0):**
- blocked: train 97.8%, test 91.6%
- interleaved: train 98.5%, test 100%
- interleaved_blocked: train 96.6%, test 91.6%

**Concern:** Interleaved slightly outperforms blocked — opposite of EGO paper finding. Likely because higher temp + learnable state_weight allows context learning under interleaved. Needs investigation.

**Not yet done:** Full 20-seed run, NeuraGEM replication, side-by-side plotting.

## Found the actual EGO Study 2 code

**Source 1 (retrieved ✓):** `figs/study2_ego_psyneulink.ipynb` — NEU-PSY-502 course notebook. Implements Coffee Shop World (CSW) task using PsyNeuLink's EMComposition. 30KB, self-contained.

**Source 2 (NOT retrieved ✗):** `Giallanza2024_ego_study2.py` — The download link from PsyNeuLink docs returns 404. File not in public GitHub repo. Generated by Sphinx documentation build only.

**Critical findings from Source 1 — my PyTorch implementation diverged in fundamental ways:**

1. **Task structure is DIFFERENT**: CSW task uses context markers 0 and 9 as sequence starters. Context 0 → [0,1,3,5,7] or [0,2,4,6,8]; Context 9 → [9,1,4,5,8] or [9,2,3,6,7]. This is NOT the Beukers et al. graphs with states 0-9 and transitions 0→1/2. States encoded as 11D one-hot.

2. **EM retrieval is fundamentally different**: PsyNeuLink's EMComposition uses 3 fields:
   - STATE (target, weight=None — NOT used in retrieval!)
   - PREVIOUS STATE (query, weight=0.5)
   - CONTEXT (query, weight=0.5, learnable projection)
   - softmax_gain=10, softmax_threshold=0.001, normalize_field_weights=True
  
3. **Context module**: TransferMechanism with Tanh, integrator_mode=True, integration_rate=0.69 (NOT MGRU with weighted gating)

4. **Loss**: BINARY_CROSS_ENTROPY (NOT MSE)

5. **Previous state buffer**: Separate mechanism maintains state_{t-1} for EM query

My PyTorch implementation used: (state, context) as dual query keys, softmax temperature, MSE loss, MGRU context module. This was a reasonable interpretation of the paper but differs from the actual PsyNeuLink implementation.
